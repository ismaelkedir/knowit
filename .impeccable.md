## Design Context

### Users
Knowit Preview serves a mixed audience:
- solo developers browsing their project memory locally
- team members browsing shared/project memory in day-to-day work
- maintainers using Preview for demos and “does it look/behave right?” validation

Primary context is quick, low-friction inspection of stored knowledge entries while working in a repo.

### Brand Personality
Credible, calm, crafted.

The UI should feel trustworthy and durable—more like a reference tool than a toy—so users believe what they’re seeing and can quickly verify “what’s stored.”

### Aesthetic Direction
Start from the existing Preview UI visual language:
- warm “paper/ink” surfaces with clear hierarchy
- restrained accent (teal) for focus/affordances, and a distinct mark color for emphasis
- high-legibility system typography and generous whitespace

Support both light and dark themes. Keep the light theme’s warmth; the dark theme should be equally credible (avoid neon-on-black).

### Design Principles
- Prefer clarity over cleverness: scanning and reading are the core tasks.
- Optimize for trust: stable layouts, predictable controls, and no “surprising” behavior.
- Fast to parse: strong hierarchy, short lines, good spacing, and readable metadata.
- Accessible by default: keyboardable UI, visible focus, and WCAG 2.2 AA-ish contrast targets.
- Motion is optional and subtle: respect `prefers-reduced-motion` and default to minimal motion.
